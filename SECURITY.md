# Security Policy

## Supported Versions

Proxy Installer 目前仅支持最新版本的安全更新。

| Version | Supported |
| ------- | --------- |
| Latest  | ✅        |
| < 1.0   | ❌        |

## Reporting a Vulnerability

我们非常重视项目的安全性。如果你发现任何安全漏洞，请按照以下步骤报告：

### 如何报告

**请勿在公开 Issue 中报告安全问题**，以免被恶意利用。

请通过以下方式之一进行报告：

1. **GitHub Private Vulnerability Reporting**（推荐）

   - 访问：https://github.com/youzhiqingw/proxy-installer/security/advisories/new
   - 填写漏洞详情，包括：
     - 漏洞类型（如：远程代码执行、信息泄露、认证绕过等）
     - 受影响的组件和版本
     - 复现步骤
     - 潜在影响
2. **直接联系维护者**

   - 如果无法使用 GitHub Advisory，请通过 GitHub Issues 创建一个**临时公开**的 Issue 并标记为 `security`，维护者会尽快将其转为私密讨论

### 报告内容

为了帮助我们更好地评估和修复漏洞，请尽量提供以下信息：

- [ ] 漏洞的详细描述
- [ ] 完整的复现步骤
- [ ] 受影响的具体版本
- [ ] 漏洞利用的潜在影响
- [ ] 修复建议（如有）

### 响应时间

- **确认收到报告**：有时间会看到
- **初步评估**：有时间会尝试解决
- **修复计划**：太难了就算了（针对已确认的漏洞）

### 处理流程

1. **接收确认** — 我们会确认收到你的报告
2. **评估验证** — 我们会验证漏洞并评估其严重程度
3. **修复开发** — 我们会开发并测试修复方案
4. **发布更新** — 修复后会发布新版本，并在 Release Notes 中说明（不透露漏洞细节）
5. **公开披露** — 在用户有时间更新后，我们会协调公开披露（如适用）

### 致谢

我们感谢所有帮助改进项目安全性的研究人员。在漏洞修复并发布后，我们会在 Release Notes 中致谢报告者（除非你选择匿名）。

## 安全最佳实践

使用 Proxy Installer 时，请遵循以下最佳实践：

- ✅ 始终使用最新版本的软件
- ✅ 妥善保管 SSH 密码和私钥
- ✅ 定期更换 VPS 凭证
- ✅ 不在公共网络环境下使用
- ✅ 启用 VPS 的防火墙和安全组
